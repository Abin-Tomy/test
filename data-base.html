E-commerce Domain: Entities & Attributes

A comprehensive reference for database design in an e-commerce platform. Each entity contains detailed attributes with recommended data types and technical justifications.

1. Customer

AttributeData TypeJustification
CustomerIDINT (PK, AUTO_INCREMENT)Unique identifier for each customer; integer for optimal indexing performance.
FirstNameVARCHAR(50)Variable length for efficient storage; 50 chars supports international names.
LastNameVARCHAR(50)Variable length string for family names across cultures.
EmailVARCHAR(254)RFC 5321 compliant maximum email length; used for authentication.
PasswordHashCHAR(60)Fixed length for BCrypt/Argon2 hashes; secure password storage.
PhoneNumberVARCHAR(20)Accommodates international formats, +, spaces, and leading zeros.
DateOfBirthDATEAge verification, targeted marketing; no time component needed.
GenderENUM('Male','Female','Other','Prefer not to say')Constrained set with privacy option; efficient storage.
AddressTEXTMulti-line addresses with varying lengths; supports international formats.
RegistrationDateDATETIMEAudit trail and customer lifecycle analysis with timestamp precision.
IsActiveBOOLEANAccount status flag; 1 bit storage for enabled/disabled state.
LastLoginDateDATETIMEUser engagement tracking and security monitoring.

2. Product

AttributeData TypeJustification
ProductIDINT (PK, AUTO_INCREMENT)Primary key for product identification; integer for join efficiency.
ProductNameVARCHAR(200)Product titles can be lengthy; supports SEO-friendly names.
DescriptionTEXTDetailed product descriptions of varying lengths; rich content support.
PriceDECIMAL(10,2)Monetary values with 2 decimal precision; avoids floating-point errors.
CategoryIDINT (FK)Foreign key to Category table; normalized design.
SupplierIDINT (FK)Links to supplier information; referential integrity.
StockQuantityINTInventory tracking; whole numbers for item counts.
WeightDECIMAL(8,3)Shipping calculations; 3 decimal places for precise measurements.
DimensionsVARCHAR(50)Length×Width×Height format; shipping and storage planning.
IsActiveBOOLEANProduct visibility control; discontinued items remain in database.
CreatedDateDATETIMEProduct lifecycle tracking; catalog management.
LastModifiedDATETIMEChange tracking for inventory and pricing updates.

3. Category

AttributeData TypeJustification
CategoryIDINT (PK, AUTO_INCREMENT)Unique category identifier for hierarchical organization.
CategoryNameVARCHAR(100)Category names with reasonable length limit.
DescriptionTEXTDetailed category descriptions for SEO and user guidance.
ParentCategoryIDINT (FK, NULL)Self-referencing for category hierarchy; NULL for root categories.
ImageURLVARCHAR(255)Category banner/icon paths; URL length accommodation.
SortOrderINTDisplay ordering control; manual arrangement capability.
IsActiveBOOLEANCategory visibility management; seasonal categories.
MetaTitleVARCHAR(60)SEO optimization; search engine title tags.
MetaDescriptionVARCHAR(160)SEO meta descriptions; search result snippets.
CreatedDateDATETIMECategory creation tracking for analytics.
ProductCountINTCached count for performance; updated via triggers.
LevelTINYINTHierarchy depth tracking; breadcrumb generation.

4. Order

AttributeData TypeJustification
OrderIDINT (PK, AUTO_INCREMENT)Unique order identifier; sequential numbering for tracking.
CustomerIDINT (FK)Links to customer record; maintains customer history.
OrderDateDATETIMEPrecise order timestamp for processing and analytics.
StatusENUM('Pending','Processing','Shipped','Delivered','Cancelled')Workflow state management; predefined status values.
TotalAmountDECIMAL(12,2)Order total with tax; higher precision for large orders.
ShippingAddressTEXTComplete delivery address; may differ from customer address.
BillingAddressTEXTBilling information; financial record keeping.
PaymentStatusENUM('Pending','Paid','Failed','Refunded')Payment state tracking; financial reconciliation.
ShippingCostDECIMAL(8,2)Separate shipping charges; cost analysis.
TaxAmountDECIMAL(10,2)Tax calculation storage; compliance reporting.
DiscountAmountDECIMAL(10,2)Applied discounts and coupons; promotion tracking.
EstimatedDeliveryDATECustomer expectation setting; logistics planning.

5. Payment

AttributeData TypeJustification
PaymentIDINT (PK, AUTO_INCREMENT)Unique payment transaction identifier.
OrderIDINT (FK)Links payment to specific order; transaction tracking.
PaymentMethodENUM('Credit Card','Debit Card','PayPal','Bank Transfer','Cash')Payment type classification; reporting and processing.
AmountDECIMAL(12,2)Payment amount with precision; financial accuracy.
CurrencyCHAR(3)ISO 4217 currency codes; international support.
TransactionIDVARCHAR(100)External payment processor reference; reconciliation.
StatusENUM('Pending','Completed','Failed','Cancelled','Refunded')Payment state management; financial workflow.
PaymentDateDATETIMETransaction timestamp; financial records.
ProcessorResponseTEXTGateway response details; troubleshooting failed payments.
LastFourDigitsCHAR(4)Card identification for customer service; PCI compliance.
ExpiryDateDATECard validity tracking; payment failure prevention.
ProcessingFeeDECIMAL(8,2)Gateway charges; profit margin calculation.

6. Review

AttributeData TypeJustification
ReviewIDINT (PK, AUTO_INCREMENT)Unique review identifier for reference and moderation.
ProductIDINT (FK)Links review to specific product; product rating aggregation.
CustomerIDINT (FK)Review author identification; prevents duplicate reviews.
RatingTINYINT1-5 star rating; small integer for efficient storage.
ReviewTitleVARCHAR(200)Review headline; summarizes customer opinion.
ReviewTextTEXTDetailed review content; variable length feedback.
ReviewDateDATETIMEReview timestamp; chronological ordering and freshness.
IsVerifiedPurchaseBOOLEANPurchase validation; authentic review identification.
HelpfulCountINTCommunity feedback on review quality; helpful sorting.
IsModeratedBOOLEANContent moderation flag; spam and abuse control.
ModeratorNotesTEXTInternal moderation comments; policy enforcement tracking.
IsPublishedBOOLEANPublication status; pending/approved review workflow.

7. Supplier

AttributeData TypeJustification
SupplierIDINT (PK, AUTO_INCREMENT)Unique supplier identification for vendor management.
CompanyNameVARCHAR(200)Business name; accommodates long corporate names.
ContactPersonVARCHAR(100)Primary contact name; relationship management.
EmailVARCHAR(254)Business communication; order and inquiry handling.
PhoneNumberVARCHAR(20)Business contact; international format support.
AddressTEXTComplete business address; shipping origin information.

PaymentTermsVARCHAR(100)Payment conditions; Net 30, COD, etc.

IsActiveBOOLEANActive relationship status; vendor lifecycle management.

8. Shipping

AttributeData TypeJustification
ShippingIDINT (PK, AUTO_INCREMENT)Unique shipping record identifier for tracking.
OrderIDINT (FK)Links to order being shipped; fulfillment tracking.
CarrierNameVARCHAR(100)Shipping company name; FedEx, UPS, DHL, etc.
TrackingNumberVARCHAR(50)Carrier tracking code; customer shipment visibility.
ShippingMethodVARCHAR(50)Service level; Standard, Express, Overnight.
ShippedDateDATETIMEDispatch timestamp; delivery calculation base.
DeliveredDateDATETIMEActual delivery time; performance metrics.
ShippingCostDECIMAL(8,2)Actual shipping charges; cost analysis.
WeightDECIMAL(8,3)Package weight; shipping cost verification.
StatusENUM('Preparing','Shipped','In Transit','Delivered','Exception')Shipment status workflow; customer communication.
DeliveryAddressTEXTComplete delivery location; GPS coordinates possible.
SignatureRequiredBOOLEANDelivery confirmation requirement; security measures.

9. Cart

AttributeData TypeJustification
CartIDINT (PK, AUTO_INCREMENT)Unique cart identifier; session and persistence management.
CustomerIDINT (FK, NULL)Cart owner; NULL for guest sessions.
SessionIDVARCHAR(128)Browser session tracking; guest cart preservation.
ProductIDINT (FK)Cart item reference; product information lookup.
QuantityINTItem count in cart; inventory checking.
PriceDECIMAL(10,2)Price at time of addition; protects against price changes.
AddedDateDATETIMECart addition timestamp; abandoned cart analysis.
UpdatedDateDATETIMELast modification time; activity tracking.
IsActiveBOOLEANCart item status; soft deletion for recovery.
ProductOptionsJSONSize, color, variants; flexible attribute storage.
DiscountAmountDECIMAL(8,2)Applied item-level discounts; promotion tracking.
SavedForLaterBOOLEANWishlist functionality; purchase intent tracking.

10. Coupon

AttributeData TypeJustification
CouponIDINT (PK, AUTO_INCREMENT)Unique coupon identifier for tracking and validation.
CouponCodeVARCHAR(50)Customer-entered code; alphanumeric promotion codes.
DescriptionVARCHAR(200)Coupon purpose and terms; customer understanding.
DiscountTypeENUM('Percentage','Fixed Amount','Free Shipping')Discount calculation method; promotion flexibility.
DiscountValueDECIMAL(8,2)Discount amount or percentage; promotion value.
MinimumAmountDECIMAL(10,2)Order minimum for coupon validity; promotion conditions.
MaximumDiscountDECIMAL(8,2)Discount cap for percentage coupons; budget control.

EndDateDATETIMEPromotion expiration; time-limited offers.
UsageLimitINTMaximum uses allowed; promotion budget control.
UsedCountINTCurrent usage tracking; availability calculation.
IsActiveBOOLEANCoupon status; enable/disable promotion control.
